1. Overview & Architecture

This demo application shows how to build a semantic product recommendation engine using EclipseStore GigaMap with integrated vector search. Instead of relying on keyword matching, products are converted into dense vector embeddings that capture their semantic meaning. A query like "wireless headphones" will find products related to audio equipment — even if they don't contain those exact words.

Key Concepts:

  • GigaMap — EclipseStore's high-performance in-memory map with built-in support for indexing, including vector indices
  • VectorIndex — An approximate nearest-neighbor (ANN) search index powered by JVector, registered directly on a GigaMap
  • Vectorizer — An abstraction for converting domain entities into numerical vectors; here implemented via LangChain4j and Ollama

The flow is straightforward: products are stored in a GigaMap. When a product is added, the ProductVectorizer converts it into a 384-dimensional vector using an Ollama embedding model. This vector is stored in the VectorIndex. When a user searches, their query is also embedded into a vector, and the index returns the most similar products via cosine similarity.

3. The Product Model

The domain entity is a simple Java record with four fields. Products are deserialized from a products.json catalog file at startup and stored in the GigaMap.

Design choice: Only name and category are used for vector embedding. Price and stock status are deliberately excluded because they don't carry semantic meaning — a $50 product and a $500 product in the same category should still be considered similar.

model/Product.java
package org.eclipse.store.demo.recommendations.model;

import com.fasterxml.jackson.annotation.JsonProperty;

public record Product(
    String name,
    String category,
    double price,
    @JsonProperty("in_stock") boolean inStock
)
{
}

The @JsonProperty("in_stock") annotation maps the snake_case JSON field to Java's camelCase convention. An example product in JSON:

{
    "name": "Wireless Headphones",
    "category": "Electronics",
    "price": 59.99,
    "in_stock": true
}

4. Product Vectorization

The ProductVectorizer is the bridge between domain objects and vector space. It extends EclipseStore's Vectorizer<Product> base class, which defines the contract for converting entities into float arrays for the VectorIndex.

How it works:

  1. A product is converted to text: "Wireless Headphones Electronics" (name + space + category)
  2. This text is passed to the Ollama embedding model via LangChain4j
  3. The model returns a 384-dimensional float array capturing the semantic meaning
  4. This vector is stored in the VectorIndex for similarity search

Vectorizer Modes

The base Vectorizer class supports two modes:

ModeisEmbedded()Behavior
Computed (default)falseVectors are computed externally (e.g., via an API). VectorIndex stores them separately.
EmbeddedtrueVectors are already stored in the entity. VectorIndex reads them on demand — no duplicate storage.

Our ProductVectorizer uses computed mode because vectors are generated by an external Ollama model and are not part of the Product record.

vectorizer/ProductVectorizer.java
package org.eclipse.store.demo.recommendations.vectorizer;

import dev.langchain4j.data.embedding.Embedding;
import dev.langchain4j.data.segment.TextSegment;
import dev.langchain4j.model.embedding.EmbeddingModel;
import org.eclipse.store.demo.recommendations.model.Product;
import org.eclipse.store.gigamap.jvector.Vectorizer;

import java.util.List;

public class ProductVectorizer extends Vectorizer<Product>
{
    private final EmbeddingModel embeddingModel;

    public ProductVectorizer(final EmbeddingModel embeddingModel)
    {
        this.embeddingModel = embeddingModel;
    }

    @Override
    public float[] vectorize(final Product product)
    {
        return this.embeddingModel.embed(toText(product)).content().vector();
    }

    @Override
    public List<float[]> vectorizeAll(final List<? extends Product> entities)
    {
        return this.embeddingModel.embedAll(
                entities.stream().map(ProductVectorizer::toText).map(TextSegment::from).toList()
            )
            .content()
            .stream()
            .map(Embedding::vector)
            .toList();
    }

    private static String toText(final Product product)
    {
        return product.name() + " " + product.category();
    }
}

Key Design Decisions

Text Representation

The toText() method concatenates name and category with a space: "Wireless Headphones Electronics". This is intentionally simple — the embedding model handles the semantic understanding. Both fields contribute to the vector: the name provides product identity and the category provides domain context, improving cross-boundary similarity matches.

Batch Vectorization

The vectorizeAll() method overrides the base class's default sequential implementation to use the embedding model's native embedAll(). This is critical for performance:

  • Reduces round-trips to the Ollama server (especially important for remote deployments)
  • Enables internal batching and parallelism within the model
  • Minimizes per-request overhead (connection setup, serialization)

At startup, when 500 products are loaded at once, this batch path avoids 500 individual HTTP calls.

5. GigaMap & Vector Index Configuration

The GigaMapConfig class wires together all infrastructure components as Spring beans. This is where the storage layer, vector indexing, and embedding model meet.

config/GigaMapConfig.java
package org.eclipse.store.demo.recommendations.config;

import dev.langchain4j.model.embedding.EmbeddingModel;
import dev.langchain4j.model.ollama.OllamaEmbeddingModel;
import org.eclipse.store.demo.recommendations.model.Product;
import org.eclipse.store.demo.recommendations.vectorizer.ProductVectorizer;
import org.eclipse.store.gigamap.jvector.VectorIndex;
import org.eclipse.store.gigamap.jvector.VectorIndexConfiguration;
import org.eclipse.store.gigamap.jvector.VectorIndices;
import org.eclipse.store.gigamap.jvector.VectorSimilarityFunction;
import org.eclipse.store.gigamap.types.GigaMap;
import org.springframework.beans.factory.annotation.Value;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

@Configuration
public class GigaMapConfig
{
    @Bean
    public EmbeddingModel embeddingModel(
        @Value("${ollama.base-url}") final String baseUrl,
        @Value("${ollama.model-name}") final String modelName
    )
    {
        return OllamaEmbeddingModel.builder()
            .baseUrl(baseUrl)
            .modelName(modelName)
            .build();
    }

    @Bean
    public ProductVectorizer productVectorizer(final EmbeddingModel embeddingModel)
    {
        return new ProductVectorizer(embeddingModel);
    }

    @Bean
    public GigaMap<Product> productGigaMap()
    {
        return GigaMap.New();
    }

    @Bean
    public VectorIndex<Product> productVectorIndex(
        final GigaMap<Product> gigaMap,
        final ProductVectorizer vectorizer
    )
    {
        final VectorIndices<Product> vectorIndices = gigaMap.index().register(VectorIndices.Category());

        final VectorIndexConfiguration config = VectorIndexConfiguration.builder()
            .dimension(384)
            .similarityFunction(VectorSimilarityFunction.COSINE)
            .build();

        return vectorIndices.add("product-embeddings", config, vectorizer);
    }
}

Walkthrough

1. Embedding Model Bean

Creates an OllamaEmbeddingModel connecting to a local Ollama instance. The all-minilm model is a lightweight sentence transformer that produces 384-dimensional embeddings — fast enough for real-time use while providing good semantic quality. This bean is shared between the ProductVectorizer (for indexing) and the ProductService (for query embedding).

2. GigaMap Bean

GigaMap.New() creates a new in-memory GigaMap instance. GigaMap serves as both the primary data store and the host for vector indices. Products added to the GigaMap are automatically passed through any registered vector indices for indexing.

3. VectorIndex Registration

This is the core integration point. Setting up a VectorIndex on a GigaMap requires three steps:

  1. Register the index category: gigaMap.index().register(VectorIndices.Category()) tells the GigaMap that vector-based indices will be used
  2. Configure the index: Specify the vector dimension (384) and similarity function (COSINE). Cosine similarity measures the angle between vectors — ideal for text embeddings where direction encodes meaning and magnitude is irrelevant
  3. Add the named index: vectorIndices.add("product-embeddings", config, vectorizer) creates a named index with the given configuration and vectorizer. The name allows multiple vector indices on the same GigaMap

6. The Product Service

The ProductService is the central orchestrator: it manages the product catalog lifecycle, handles similarity search, and provides paginated listing. This is where all the pieces come together.

service/ProductService.java
package org.eclipse.store.demo.recommendations.service;

import com.fasterxml.jackson.core.type.TypeReference;
import com.fasterxml.jackson.databind.ObjectMapper;
import dev.langchain4j.model.embedding.EmbeddingModel;
import jakarta.annotation.PostConstruct;
import org.eclipse.store.demo.recommendations.model.Product;
import org.eclipse.store.gigamap.jvector.VectorIndex;
import org.eclipse.store.gigamap.jvector.VectorSearchResult;
import org.eclipse.store.gigamap.types.GigaMap;
import org.slf4j.Logger;
import org.slf4j.LoggerFactory;
import org.springframework.core.io.ClassPathResource;
import org.springframework.stereotype.Service;

import java.io.IOException;
import java.io.InputStream;
import java.util.ArrayList;
import java.util.List;
import java.util.Random;

@Service
public class ProductService
{
    private static final Logger LOG                   = LoggerFactory.getLogger(ProductService.class);
    private static final int    INITIAL_PRODUCT_COUNT = 500;

    private final GigaMap<Product>      gigaMap;
    private final VectorIndex<Product>  vectorIndex;
    private final EmbeddingModel        embeddingModel;
    private final List<Product>         availableProducts;
    private final Random                random = new Random();

    public ProductService(
        final GigaMap<Product> gigaMap,
        final VectorIndex<Product> vectorIndex,
        final EmbeddingModel embeddingModel
    )
    {
        this.gigaMap           = gigaMap;
        this.vectorIndex       = vectorIndex;
        this.embeddingModel    = embeddingModel;
        this.availableProducts = loadProductCatalog();
    }

    @PostConstruct
    public void init()
    {
        if(!this.gigaMap.isEmpty())
        {
            LOG.info("GigaMap already contains {} products, skipping initialization.",
                this.gigaMap.size());
            return;
        }

        final int count = Math.min(INITIAL_PRODUCT_COUNT, this.availableProducts.size());
        LOG.info("Initializing GigaMap with {} products...", count);

        this.addRandomProducts(count);

        LOG.info("Initialization complete. GigaMap contains {} products.",
            this.gigaMap.size());
    }

    public List<Product> addRandomProducts(final int count)
    {
        final List<Product> added = new ArrayList<>(count);
        for(int i = 0; i < count && !this.availableProducts.isEmpty(); i++)
        {
            added.add(this.availableProducts.remove(
                this.random.nextInt(this.availableProducts.size())));
        }
        this.gigaMap.addAll(added);
        return added;
    }

    public List<SearchResult> search(final String query, final int k)
    {
        final float[] queryVector = this.embeddingModel.embed(query).content().vector();
        final VectorSearchResult<Product> result = this.vectorIndex.search(queryVector, k);

        final List<SearchResult> results = new ArrayList<>();
        for(final ScoredSearchResult.Entry<Product> entry : result)
        {
            results.add(new SearchResult(entry.entity(), entry.score()));
        }
        return results;
    }

    public PageResult list(final int page, final int size)
    {
        final long total = this.gigaMap.size();
        final int  skip  = page * size;

        final List<Product> products = this.gigaMap.query().stream()
            .skip(skip).limit(size).toList();

        return new PageResult(products, total, page, size);
    }

    private static List<Product> loadProductCatalog()
    {
        try(final InputStream is = new ClassPathResource("products.json").getInputStream())
        {
            return new ObjectMapper().readValue(is, new TypeReference<>() {});
        }
        catch(final IOException e)
        {
            throw new RuntimeException("Failed to load products.json", e);
        }
    }

    public record SearchResult(Product product, float score) {}

    public record PageResult(List<Product> products, long total, int page, int size) {}
}

Walkthrough

Initialization & Catalog Loading

The constructor loads the full product catalog from products.json into a mutable list. The @PostConstruct init() method then seeds the GigaMap with up to 500 random products. The idempotency check (gigaMap.isEmpty()) ensures that if the application restarts with a persistent GigaMap, existing data is preserved.

Adding Products

addRandomProducts() randomly selects products from the remaining catalog pool and adds them via gigaMap.addAll(). This single call triggers the GigaMap to automatically pass each product through the registered ProductVectorizer, compute its embedding, and insert it into the VectorIndex. No manual indexing step is needed.

Semantic Search

The search method is remarkably concise — just three lines of logic:

  1. Embed the query: embeddingModel.embed(query).content().vector() converts the user's natural-language query into the same 384-dimensional vector space as the products
  2. Search the index: vectorIndex.search(queryVector, k) performs approximate nearest-neighbor search, returning the top-k most similar products
  3. Map results: Each ScoredSearchResult.Entry contains the matched entity and its cosine similarity score (0.0 to 1.0)

Response Records

Two inner records provide clean response structures:

  • SearchResult(product, score) — pairs a matched product with its similarity score
  • PageResult(products, total, page, size) — standard pagination envelope

7. The REST Controller

The ProductController exposes three endpoints under /products:

MethodPathDescriptionKey Parameters
GET /products Paginated listing of all products page (default: 0), size (default: 20)
GET /products/search Semantic similarity search query (required), k (default: 5)
POST /products/add Add random products from catalog count (default: 1)
controller/ProductController.java
package org.eclipse.store.demo.recommendations.controller;

import org.eclipse.store.demo.recommendations.model.Product;
import org.eclipse.store.demo.recommendations.service.ProductService;
import org.eclipse.store.demo.recommendations.service.ProductService.PageResult;
import org.eclipse.store.demo.recommendations.service.ProductService.SearchResult;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.PostMapping;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.RequestParam;
import org.springframework.web.bind.annotation.RestController;

import java.util.List;

@RestController
@RequestMapping("/products")
public class ProductController
{
    private final ProductService productService;

    public ProductController(final ProductService productService)
    {
        this.productService = productService;
    }

    @GetMapping
    public PageResult list(
        @RequestParam(defaultValue = "0") final int page,
        @RequestParam(defaultValue = "20") final int size
    )
    {
        return this.productService.list(page, size);
    }

    @GetMapping("/search")
    public List<SearchResult> search(
        @RequestParam final String query,
        @RequestParam(defaultValue = "5") final int k
    )
    {
        return this.productService.search(query, k);
    }

    @PostMapping("/add")
    public List<Product> addRandom(@RequestParam(defaultValue = "1") final int count)
    {
        return this.productService.addRandomProducts(count);
    }
}

Note: The controller is deliberately thin — it contains no business logic, only parameter binding and delegation. All the heavy lifting (embedding, searching, GigaMap interaction) lives in the ProductService. A Swagger UI is also auto-configured at /swagger-ui.html for interactive testing.

8. Running the Demo

Prerequisites

  1. Java 17+ (required by Spring Boot 4)
  2. Ollama installed and running locally
  3. Pull the embedding model: ollama pull all-minilm

Start the Application

# From the project root
mvn -pl demos/product-recommendations spring-boot:run

On first startup, the service loads products from products.json, seeds 500 into the GigaMap (each being vectorized via Ollama), and starts the REST API on port 8080.

Try the API

Search for products

curl "http://localhost:8080/products/search?query=wireless+headphones&k=5"

Example response:

[
  {
    "product": {
      "name": "Bluetooth Over-Ear Headphones",
      "category": "Electronics",
      "price": 79.99,
      "in_stock": true
    },
    "score": 0.892
  },
  ...
]

List products (paginated)

curl "http://localhost:8080/products?page=0&size=10"

Add more products

curl -X POST "http://localhost:8080/products/add?count=50"

Swagger UI

Open http://localhost:8080/swagger-ui.html in your browser for interactive API exploration with "Try it out" enabled by default.
