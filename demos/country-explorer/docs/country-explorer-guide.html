1. Overview & Architecture

This demo application shows how to build a geospatial country exploration API using EclipseStore GigaMap with integrated bitmap and spatial indices. Instead of relying on a traditional database with SQL queries, all country data lives in an in-memory GigaMap that supports proximity searches, bounding-box queries, hemisphere filtering, and more — all backed by high-performance bitmap indices.

Key Concepts:

  • GigaMap — EclipseStore's high-performance in-memory map with built-in support for multiple index types, including bitmap and spatial indices
  • SpatialIndexer — An index that maps entities to latitude/longitude coordinates, enabling radius searches, bounding-box queries, and hemisphere filtering
  • IndexerString — A bitmap index on string fields, enabling efficient exact-match lookups by continent, country name, or ISO code

The flow is straightforward: on startup, country data is loaded from a bundled countries.json resource into the GigaMap. Four bitmap indices are registered — a spatial index for latitude/longitude and three string indices for continent, name, and alpha-2 code. The REST API then exposes a rich set of query endpoints that leverage these indices for sub-millisecond lookups across proximity, containment, and equality queries.

3. The Country Model

The domain entity is a Java record with nine fields representing a country's identity, geographic coordinates, and basic demographic data. Countries are deserialized from a countries.json catalog file at startup and stored in the GigaMap.

Design choice: The record uses the capital city's coordinates for latitude and longitude. This is a deliberate simplification — representing each country as a single point makes spatial queries (nearby, distance) straightforward, while still providing meaningful results for geographic exploration.

4. Country Indices

The CountryIndices class is a central registry of all GigaMap indices used by the application. It holds four singleton index instances that are shared between configuration (where they are registered) and the service layer (where they are used for querying).

Index types at a glance:

  • SpatialIndexer (LOCATION) — Maps countries to lat/lon coordinates. Supports radius search, bounding-box containment, hemisphere filtering, and latitude-band queries
  • IndexerString (CONTINENT, NAME, ALPHA2) — Bitmap indices on string fields for efficient exact-match lookups. GigaMap evaluates these using bitwise operations, making them extremely fast even for large datasets
index/CountryIndices.java
package org.eclipse.store.demo.countries.index;

import org.eclipse.store.demo.countries.model.Country;
import org.eclipse.store.gigamap.types.IndexerString;

public final class CountryIndices
{
    public static final LocationIndex LOCATION = new LocationIndex();

    public static final IndexerString<Country> CONTINENT = new IndexerString.Abstract<>()
    {
        @Override
        protected String getString(final Country country)
        {
            return country.continent();
        }
    };

    public static final IndexerString<Country> NAME = new IndexerString.Abstract<>()
    {
        @Override
        protected String getString(final Country country)
        {
            return country.name();
        }
    };

    public static final IndexerString<Country> ALPHA2 = new IndexerString.Abstract<>()
    {
        @Override
        protected String getString(final Country country)
        {
            return country.alpha2();
        }
    };

    private CountryIndices()
    {
        // no instances
    }
}

Walkthrough

LOCATION — Spatial Index

The LocationIndex instance (detailed in the next section) extracts latitude and longitude from each country. It enables the richest set of queries: radius search, bounding-box containment, hemisphere filtering, and latitude-band queries (e.g. tropical zone). This single index powers six of the ten API endpoints.

CONTINENT — String Bitmap Index

Extracts the continent() field. Used by the /countries/continent/{name} endpoint to return all countries on a given continent. The bitmap index makes this a constant-time bitwise operation regardless of dataset size.

NAME — String Bitmap Index

Extracts the name() field. Available for exact-match country name lookups.

ALPHA2 — String Bitmap Index

Extracts the alpha2() field (e.g. "DE", "US"). This is the primary lookup index — the findByAlpha2() method is used by seven endpoints that need to resolve a country code to a Country object before performing further queries.

5. The Spatial Location Index

The LocationIndex extends GigaMap's SpatialIndexer.Abstract to map each Country to its geographic coordinates. This is the bridge between the domain model and the spatial query engine.

index/LocationIndex.java
package org.eclipse.store.demo.countries.index;

import org.eclipse.store.demo.countries.model.Country;
import org.eclipse.store.gigamap.types.SpatialIndexer;

public class LocationIndex extends SpatialIndexer.Abstract<Country>
{
    @Override
    protected Double getLatitude(final Country country)
    {
        return country.latitude();
    }

    @Override
    protected Double getLongitude(final Country country)
    {
        return country.longitude();
    }
}

How SpatialIndexer works: By implementing just two methods — getLatitude() and getLongitude() — you unlock a rich set of spatial query capabilities inherited from SpatialIndexer.Abstract:

  • near(lat, lon, radiusKm) — finds candidates within a circle
  • withinRadius(lat, lon, radiusKm) — precise haversine-distance predicate filter
  • withinBox(minLat, maxLat, minLon, maxLon) — bounding-box containment
  • latitudeAbove(threshold) / latitudeBelow(threshold) — hemisphere filtering
  • longitudeAbove(threshold) / longitudeBelow(threshold) — east/west hemisphere
  • latitudeBetween(min, max) — latitude-band queries (e.g. tropical zone)

Two-Phase Spatial Query Pattern

GigaMap spatial queries typically use a two-phase approach:

  1. Candidate selection — near(lat, lon, radiusKm) uses the bitmap index to quickly identify countries that might be within the radius (bounding-box approximation)
  2. Precise filtering — withinRadius(lat, lon, radiusKm) applies the haversine formula to each candidate, removing false positives that fall outside the circular radius but inside the rectangular bounding box

This two-phase pattern is visible in the CountryService.nearCoordinates() method and combines the speed of bitmap operations with the precision of great-circle distance calculations.

6. GigaMap Configuration

The GigaMapConfig class wires together the GigaMap and all indices as Spring beans. This is where the storage layer and indexing infrastructure are assembled.

config/GigaMapConfig.java
package org.eclipse.store.demo.countries.config;

import org.eclipse.store.demo.countries.index.LocationIndex;
import org.eclipse.store.demo.countries.index.CountryIndices;
import org.eclipse.store.demo.countries.model.Country;
import org.eclipse.store.gigamap.types.GigaMap;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

@Configuration
public class GigaMapConfig
{
    @Bean
    public GigaMap<Country> countryGigaMap()
    {
        final GigaMap<Country> gigaMap = GigaMap.New();
        gigaMap.index().bitmap().addAll(
            CountryIndices.LOCATION,
            CountryIndices.CONTINENT,
            CountryIndices.NAME,
            CountryIndices.ALPHA2
        );
        return gigaMap;
    }

    @Bean
    public LocationIndex locationIndex()
    {
        return CountryIndices.LOCATION;
    }
}

Walkthrough

1. GigaMap Bean

GigaMap.New() creates a new in-memory GigaMap instance. GigaMap serves as both the primary data store and the host for all indices. Countries added to the GigaMap are automatically indexed by every registered indexer.

2. Bitmap Index Registration

This is the core setup step. A single call to gigaMap.index().bitmap().addAll() registers all four indices at once:

  1. LOCATION — the SpatialIndexer that enables geographic queries. Despite being a spatial index, it is registered as a bitmap index because GigaMap's spatial queries are implemented on top of the bitmap index infrastructure
  2. CONTINENT — string bitmap on the continent field
  3. NAME — string bitmap on the country name
  4. ALPHA2 — string bitmap on the ISO alpha-2 code

Once registered, any call to gigaMap.addAll() or gigaMap.add() automatically updates all four indices. No manual re-indexing is needed.

3. LocationIndex Bean

The LocationIndex singleton from CountryIndices.LOCATION is also exposed as a standalone Spring bean. This allows the CountryService to call spatial query methods (near(), withinBox(), etc.) directly on the index instance, rather than going through the GigaMap's generic index registry.

7. The Country Service

The CountryService is the central orchestrator: it manages the data lifecycle, provides all query operations, and computes distances. This is where all the pieces come together.

service/CountryService.java
package org.eclipse.store.demo.countries.service;

import com.fasterxml.jackson.core.type.TypeReference;
import com.fasterxml.jackson.databind.ObjectMapper;
import jakarta.annotation.PostConstruct;
import org.eclipse.store.demo.countries.index.LocationIndex;
import org.eclipse.store.demo.countries.index.CountryIndices;
import org.eclipse.store.demo.countries.model.Country;
import org.eclipse.store.gigamap.types.GigaMap;
import org.eclipse.store.gigamap.types.SpatialIndexer;
import org.slf4j.Logger;
import org.slf4j.LoggerFactory;
import org.springframework.core.io.ClassPathResource;
import org.springframework.stereotype.Service;

import java.io.IOException;
import java.io.InputStream;
import java.util.Comparator;
import java.util.List;

@Service
public class CountryService
{
    private static final Logger LOG = LoggerFactory.getLogger(CountryService.class);

    private final GigaMap<Country> gigaMap;
    private final LocationIndex     locationIndex;

    public CountryService(
        final GigaMap<Country> gigaMap,
        final LocationIndex locationIndex
    )
    {
        this.gigaMap       = gigaMap;
        this.locationIndex = locationIndex;
    }

    @PostConstruct
    public void init()
    {
        if(!this.gigaMap.isEmpty())
        {
            LOG.info("GigaMap already contains {} countries, skipping initialization.",
                this.gigaMap.size());
            return;
        }

        final List<Country> countries = loadCountries();
        this.gigaMap.addAll(countries);
        LOG.info("Loaded {} countries into GigaMap.", this.gigaMap.size());
    }

    // ... query methods shown below ...

    private static List<Country> loadCountries()
    {
        try(final InputStream is = new ClassPathResource("countries.json").getInputStream())
        {
            return new ObjectMapper().readValue(is, new TypeReference<>() {});
        }
        catch(final IOException e)
        {
            throw new RuntimeException("Failed to load countries.json", e);
        }
    }

    public record NeighbourResult(Country country, double distanceKm) {}
    public record DistanceResult(Country from, Country to, double distanceKm) {}
    public record PageResult(List<Country> countries, long total, int page, int size) {}
}

Initialization & Data Loading

The constructor receives the GigaMap and LocationIndex beans via Spring injection. The @PostConstruct init() method loads all countries from countries.json via Jackson and inserts them into the GigaMap with a single addAll() call. The idempotency check (gigaMap.isEmpty()) ensures that if the application restarts with a persistent GigaMap, existing data is preserved.

Query Methods

Paginated Listing

public PageResult list(final int page, final int size)
{
    final long total = this.gigaMap.size();
    final List<Country> countries = this.gigaMap.query()
        .stream()
        .sorted(Comparator.comparing(Country::name))
        .skip((long) page * size)
        .limit(size)
        .toList();
    return new PageResult(countries, total, page, size);
}

Uses gigaMap.query() with no predicates to retrieve all countries, then applies Java Stream operations for sorting and pagination. The PageResult record wraps the result with metadata for clients to navigate pages.

Alpha-2 Lookup

public Country findByAlpha2(final String alpha2)
{
    return this.gigaMap.query(CountryIndices.ALPHA2.is(alpha2.toUpperCase()))
        .findFirst()
        .orElse(null);
}

The ALPHA2.is() call creates a bitmap predicate that resolves in constant time. The toUpperCase() normalization makes the lookup case-insensitive. This method is the building block for many other queries that need to resolve a country code first.

Radius Search (by Country)

public List<NeighbourResult> nearby(final String alpha2, final double radiusKm)
{
    final Country origin = this.findByAlpha2(alpha2);
    if(origin == null) return List.of();

    return this.nearCoordinates(origin.latitude(), origin.longitude(), radiusKm)
        .stream()
        .filter(r -> !r.country().alpha2().equals(origin.alpha2()))
        .toList();
}

Resolves the origin country by alpha-2 code, delegates to nearCoordinates() for the actual spatial query, then filters out the origin country from the results.

Radius Search (by Coordinates)

This is the core spatial query method — it demonstrates the two-phase pattern:

  1. Phase 1 — Candidate selection: locationIndex.near(lat, lon, radiusKm) uses the bitmap index to quickly identify countries within a rectangular bounding box
  2. Phase 2 — Precise filtering: locationIndex.withinRadius(lat, lon, radiusKm) applies the haversine formula to each candidate, keeping only those truly within the circular radius
public List<NeighbourResult> nearCoordinates(
    final double lat, final double lon, final double radiusKm)
{
    return this.gigaMap.query(this.locationIndex.near(lat, lon, radiusKm))
        .stream()
        .filter(this.locationIndex.withinRadius(lat, lon, radiusKm))
        .map(c -> new NeighbourResult(
            c,
            Math.round(SpatialIndexer.haversineDistance(
                lat, lon, c.latitude(), c.longitude()) * 10.0) / 10.0
        ))
        .sorted(Comparator.comparingDouble(NeighbourResult::distanceKm))
        .toList();
}

K-Nearest Neighbours

public List<NeighbourResult> kNearest(final String alpha2, final int k)
{
    final Country origin = this.findByAlpha2(alpha2);
    if(origin == null) return List.of();

    final double lat = origin.latitude();
    final double lon = origin.longitude();
    return this.gigaMap.query()
        .stream()
        .filter(c -> !c.alpha2().equals(origin.alpha2()))
        .map(c -> new NeighbourResult(c,
            Math.round(SpatialIndexer.haversineDistance(
                lat, lon, c.latitude(), c.longitude()) * 10.0) / 10.0))
        .sorted(Comparator.comparingDouble(NeighbourResult::distanceKm))
        .limit(k)
        .toList();
}

Unlike nearby(), this method does not use a fixed radius. It computes the haversine distance to every other country, sorts by distance, and takes the top k. This brute-force approach is perfectly acceptable for the ~250 countries in the dataset.

Continent Filtering

public List<Country> continent(final String name)
{
    return this.gigaMap.query(CountryIndices.CONTINENT.is(name))
        .stream()
        .sorted(Comparator.comparing(Country::name))
        .toList();
}

Uses the CONTINENT bitmap index for a fast exact-match filter. The bitmap index resolves the matching countries in a single bitwise operation.

Bounding-Box Search

public List<Country> withinBox(
    final double minLat, final double maxLat,
    final double minLon, final double maxLon)
{
    return this.gigaMap.query(this.locationIndex.withinBox(minLat, maxLat, minLon, maxLon))
        .stream()
        .sorted(Comparator.comparing(Country::name))
        .toList();
}

Delegates directly to the spatial index's withinBox() predicate. This is useful for map-viewport queries where a user is looking at a specific region on a map.

Hemisphere & Tropical Zone

public List<Country> hemisphere(final String name)
{
    return switch(name.toLowerCase())
    {
        case "north" -> this.gigaMap.query(this.locationIndex.latitudeAbove(0.0))
            .stream().sorted(Comparator.comparing(Country::name)).toList();
        case "south" -> this.gigaMap.query(this.locationIndex.latitudeBelow(0.0))
            .stream().sorted(Comparator.comparing(Country::name)).toList();
        case "east"  -> this.gigaMap.query(this.locationIndex.longitudeAbove(0.0))
            .stream().sorted(Comparator.comparing(Country::name)).toList();
        case "west"  -> this.gigaMap.query(this.locationIndex.longitudeBelow(0.0))
            .stream().sorted(Comparator.comparing(Country::name)).toList();
        default -> List.of();
    };
}

public List<Country> tropical()
{
    return this.gigaMap.query(this.locationIndex.latitudeBetween(-23.5, 23.5))
        .stream()
        .sorted(Comparator.comparing(Country::name))
        .toList();
}

The hemisphere() method uses a Java switch expression to map hemisphere names to the appropriate spatial predicate. The tropical() method uses latitudeBetween(-23.5, 23.5) to find countries between the Tropics of Cancer and Capricorn. Both demonstrate how GigaMap's spatial predicates compose naturally with the query API.

Distance Calculation

public DistanceResult distance(final String fromAlpha2, final String toAlpha2)
{
    final Country from = this.findByAlpha2(fromAlpha2);
    final Country to   = this.findByAlpha2(toAlpha2);
    if(from == null || to == null) return null;

    final double km = Math.round(
        SpatialIndexer.haversineDistance(
            from.latitude(), from.longitude(),
            to.latitude(), to.longitude()) * 10.0
    ) / 10.0;
    return new DistanceResult(from, to, km);
}

Uses GigaMap's built-in SpatialIndexer.haversineDistance() utility to compute the great-circle distance between two capital cities. The result is rounded to one decimal place for clean output.

Response Records

Three inner records provide clean response structures:

  • NeighbourResult(country, distanceKm) — pairs a country with its distance from a reference point (used by nearby, nearCoordinates, and kNearest)
  • DistanceResult(from, to, distanceKm) — the two countries and the distance between them
  • PageResult(countries, total, page, size) — standard pagination envelope

controller/CountryController.java
package org.eclipse.store.demo.countries.controller;

import org.eclipse.store.demo.countries.model.Country;
import org.eclipse.store.demo.countries.service.CountryService;
import org.eclipse.store.demo.countries.service.CountryService.DistanceResult;
import org.eclipse.store.demo.countries.service.CountryService.NeighbourResult;
import org.eclipse.store.demo.countries.service.CountryService.PageResult;
import org.springframework.http.ResponseEntity;
import org.springframework.web.bind.annotation.*;

import java.util.List;

@RestController
@RequestMapping("/countries")
public class CountryController
{
    private final CountryService countryService;

    public CountryController(final CountryService countryService)
    {
        this.countryService = countryService;
    }

    @GetMapping
    public PageResult list(
        @RequestParam(defaultValue = "0") final int page,
        @RequestParam(defaultValue = "20") final int size)
    {
        return this.countryService.list(page, size);
    }

    @GetMapping("/{alpha2}")
    public ResponseEntity<Country> findByAlpha2(@PathVariable final String alpha2)
    {
        final Country country = this.countryService.findByAlpha2(alpha2);
        return country != null
            ? ResponseEntity.ok(country)
            : ResponseEntity.notFound().build();
    }

    @GetMapping("/nearby")
    public List<NeighbourResult> nearby(
        @RequestParam final String alpha2,
        @RequestParam(defaultValue = "1000") final double radiusKm)
    {
        return this.countryService.nearby(alpha2, radiusKm);
    }

    @GetMapping("/near")
    public List<NeighbourResult> nearCoordinates(
        @RequestParam final double lat,
        @RequestParam final double lon,
        @RequestParam(defaultValue = "1000") final double radiusKm)
    {
        return this.countryService.nearCoordinates(lat, lon, radiusKm);
    }

    @GetMapping("/nearest")
    public List<NeighbourResult> kNearest(
        @RequestParam final String alpha2,
        @RequestParam(defaultValue = "5") final int k)
    {
        return this.countryService.kNearest(alpha2, k);
    }

    @GetMapping("/continent/{name}")
    public List<Country> continent(@PathVariable final String name)
    {
        return this.countryService.continent(name);
    }

    @GetMapping("/within-box")
    public List<Country> withinBox(
        @RequestParam final double minLat,
        @RequestParam final double maxLat,
        @RequestParam final double minLon,
        @RequestParam final double maxLon)
    {
        return this.countryService.withinBox(minLat, maxLat, minLon, maxLon);
    }

    @GetMapping("/hemisphere")
    public List<Country> hemisphere(@RequestParam final String hemisphere)
    {
        return this.countryService.hemisphere(hemisphere);
    }

    @GetMapping("/tropical")
    public List<Country> tropical()
    {
        return this.countryService.tropical();
    }

    @GetMapping("/distance")
    public ResponseEntity<DistanceResult> distance(
        @RequestParam final String from,
        @RequestParam final String to)
    {
        final DistanceResult result = this.countryService.distance(from, to);
        return result != null
            ? ResponseEntity.ok(result)
            : ResponseEntity.notFound().build();
    }
}

Note: The controller is deliberately thin — it contains no business logic, only parameter binding and delegation. All the heavy lifting (spatial queries, distance calculations, GigaMap interaction) lives in the CountryService. Two endpoints (findByAlpha2 and distance) return ResponseEntity to produce proper 404 Not Found responses when a country code is invalid. A Swagger UI is also auto-configured at /swagger-ui.html for interactive testing.
